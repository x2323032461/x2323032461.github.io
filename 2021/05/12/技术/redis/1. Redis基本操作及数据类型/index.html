Redis五大数据类型

Redis 是一个开源（BSD许可）的，内存中的数据结构存储系统，它可以用作数据库、缓存和消息中间件。 它支持多种类型的数据结构，如 字符串（strings）， 散列（hashes）， 列表（lists）， 集合（sets）， 有序集合（sorted sets） 与范围查询， bitmaps， hyperloglogs 和 地理空间（geospatial） 索引半径查询。 Redis 内置了 复制（replication），LUA脚本（Lua scripting）， LRU驱动事件（LRU eviction），事务（transactions） 和不同级别的 磁盘持久化（persistence）， 并通过 Redis哨兵（Sentinel）和自动 分区（Cluster）提供高可用性（high availability）。

2. List（列表，值可以重复）

所有的list命令都是 L 开头的

redis的list中可以存储重复的值

小结

  • 实际上是一个链表
  • 如果key不存在，则会创建一个新的链表
  • 如果key存在，则会新增内容
  • 如果移除了所有的值，即一个空链表，也表示不存在
  • 在两边插入或者改动效率最高，修改中间元素，效率相对会低一点

可用作：栈、队列、阻塞队列

  • 队列：使用lpush 添加数据，rpop拿取数据
  • 栈：使用lpush 添加数据，lpop拿取数据

3. Set（集合，无序不重复集合）

所有的set命令都是 S 开头的

应用

微博，将用户所有关注的人放在一个set集合中，将他的粉丝也防在一个set集合中

共同关注，共同爱好，二度好友（好友推荐），即可实现

4. Hash（可想象成元素为map集合）

所有的set命令都是 H 开头的

应用

尤其是用户信息或经常变动的信息的保存

因此hash更适合对象的存储

5. Zset（有序集合）

在set的基础上，增加了一个值，分数值 score

应用

存储班级成绩，工资表排序，播放量排行榜实现

普通消息：1，重要消息：2，带权重进行判断

1. Geospatial地理位置 命令以geo开头

Redis的Geo功能可以推算地理位置的信息，推算两地之间的距离，方圆几里的人

可获取数据位置：https://jingweidu.bmcx.com/

官方介绍文档：https://www.redis.net.cn/order/3685.html

1.0. geo底层原理

geo底层其实就是zset，因此可以使用zset的命令操作geo的内容

1.1. geoadd 添加地理位置

geoadd key 经度 维度 名称

  • 有效的经度从-180度到180度。
  • 有效的纬度从-85.05112878度到85.05112878度。
  • 当坐标位置超出上述指定范围时，该命令将会返回一个错误。

1.3. geodist 获取两地距离

  • m 表示单位为米。

1.4. georadius 以给定的经纬度为中心， 找出某一半径内的元素

可用作查找附近的人，通过半径查询

2. Hyperloglog

2.1 说明

基数：找不重复的元素

Hyperloglog存在约0.81%的错误率

A {1,3,5,7,8,7}

基数为： 5

简介

Hyperloglog用来做基数统计的算法

优点：

  • 占用的内存是固定的，存储2^64不同的元素，只需要费12KB内存

网页的UV（访问统计）（一个人访问该网站多次，但是还是算作一个人）

传统的方式，set保存用户的id，然后就可以通过统计set中的元素数量作为标准判断

这个方式如果保存大量的用户id，就会比较麻烦！我们的目的是为了计数而不是用来保存用户id；
